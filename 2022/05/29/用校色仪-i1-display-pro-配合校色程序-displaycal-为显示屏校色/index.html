用校色仪 i1 Display Pro 配合校色程序 DisplayCAL 为显示屏校色

发表于 2022-05-29 更新于 2022-05-28 分类于 杂项 Valine：

这几天折腾了一下校色，在这里简单记录一下大致的流程：

确保 Windows 系统的颜色校准已关闭

打开 Windows 的“颜色管理”，在“高级”选项卡中点击“更改系统默认值”

将“高级”选项卡中“显示器校准”中的“使用 Windows 显示器校准”一项的打钩取消，即关闭 Windows 系统的颜色校准，如果本身就未打钩则可无需进一步处理

安装与配置校色程序 DisplayCAL 和 ArgyllCMS

使用 exe 可执行文件安装并配置

前往DisplayCAL 官方网页，点击后会跳转到 sourceforge 进行下载

安装过程中保证选中这项，即使用 DisplayCAL 而非 Windows 系统进行颜色校准；DisplayCAL 安装在 C:\Program Files (x86)\DisplayCAL，配置文件在 %AppData%\DisplayCAL

打开 DisplayCAL，提示需要安装校色程序 ArgyllCMS，点击 Download 进行安装

DisplayCAL 会自行联网下载并安装 ArgyllCMS，ArgyllCMS 安装在 C:\ProgramData\ArgyllCMS

调整语言至简体中文

检查手边的校色仪型号，租用的校色仪一般都是蜘蛛（Spyder）系列和爱色丽（X-Rite）系列。如果用的是蜘蛛系列的话，需要手动安装 ArgyllCMS 驱动程序后才可以检测到连接到电脑上的校色仪

手动安装时官方也给出了说明，如果用的是爱色丽（X-Rite）系列，则无需安装 ArgyllCMS 驱动程序

ArgyllCMS 驱动程序安装在 %AppData%\DisplayCAL\dl\Argyll_V2.1.2

备注：使用 scoop 安装并配置

如果使用 scoop 进行安装，则只需在 Powershell 中执行命令 scoop install displaycal，即可完成 DisplayCAL 和 ArgyllCMS 的安装，文件均安装在 scoop 文件夹内

其他操作与之前相同，由于已经安装 ArgyllCMS，DisplayCAL 开启的时候不会有提示，直接调整语言即可使用。但 scoop 版本有一个致命缺陷，无法安装 ArgyllCMS 驱动程序，即蜘蛛系列的校色仪无法识别也无法使用。下载驱动程序时会出现报错 Key doesn't exist: "There is no item named u'Argyll_V2.3.0_UBS_driver_install.exe' in the archive"，原因大概是和 ArgyllCMS 驱动程序的版本有关，解决方法未知

校色设置

打开选项内的“显示高级选项”
